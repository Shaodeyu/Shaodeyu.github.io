图解HTTP。学习笔记

发表于 2016-06-19   |  

在1989年3月，互联网还只是属于少数人，在这一互联网的黎明期，HTTP诞生了。

天才的设想远远都超出人的想象，他们不像暴发户那样，想着要不我们就少数人玩这个互联网新玩意就好了。
于是他们提出了WWW的构建技术。
用于SGML（standard Generalized Markup Language）的文本标记语言HTML。
作为文档传输协议的HTTP。
指定文档所在地址的URL（Uniform Resource Locater）.

今天我们要将的是HTTP，拿我们就必须先了解一下TCP/IP协议族。
现在我们通常使用的网络（包括互联网）都是在TCP／IP协议族的基础上运作的，而HTTP属于它内部的一个子集。

我想跟你讲话，就要用大家都相同的语言，比如中文。
同理，计算机要想与网络设备互相通信，双方也必须基于相同的方法。
比如，如和探测到通信目标，由哪一边先发起通信。使用哪种语言进行通信，怎样结束通信等规则都需要事先确定。
把互联网相关的协议集合起来总称就为TCP/IP。

汽车是由各种不同的部件组合起来在一起的，这样有很多的好处。当我们的轮胎坏的时候，我们只需要拿出单独的轮胎来修就可以了。天才们设计TCP／IP也是分层的。
TCP/IP分为四个层：应用层，传输层，网络层，和数据链路层。
应用层
应用层决定了向用户提供应用服务时通信的活动。
比如FTP和DNS服务就是这两类。
当然今天我们的HTTP主角也是在这一层的哦。
传输层
传输层对应上层应用层，提供处于网络连接中的两台计算机之间的数据传输。
传输层有两个不同的协议。
TCP（）和UDP（User Data Protocol，用户数据报协议）。
网络层
网络层用来处理在网络上流动的数据包。
链路层
用来处理连接网络的硬件部分。包括控制操作系统。硬件的设备驱动。NIC（Network Interface Card，其实就是网卡啦。）等物理部分。

每个人都有几个很要好的朋友。
对于HTTP，当然也是这样。在TCP/IP协议当中。
HTTP和IP，TCP，DNS密不可分。
负责传输协议的IP协议。
按层次分，IP（Inter Protocol）网际协议位于应用层。几乎所有的使用网络系统都会用到IP协议。
TCP／IP协议中的IP占据了协议的一半的名字，可见其重要性。IP和IP协议可是不同的喔，亲。IP协议的作用就是把各种数据包传送给对方。而要保证确实传送到对方那里，就要满足各类条件。其中两个重要的条件就是IP地址和MAC地址（Media Access Control Address）（此MAC不是Macbook的mac）.
使用ARP协议凭借MAC地址进行通信。
IP间的通信依赖MAC地址。在进行通信的时候，通常都是经过多台电脑多台网络设备中转的。那么问题来啦？中转需要什么样的协议？这个时候就用到ARP（Address Resolution Protocol）协议了。在进行中转的时候，会利用下一站中转设备的MAC地址来搜索下一个中转目标。ARP是一种用来解析地址的协议，根据通信方的IP地址就可以反查出对应的MAC地址。

其实没有人能够全面的掌握互联网传输的传输状况。
就像我们上淘宝买东西的时候，我们看快递只能够知道到了哪个地方，下一站在那里，而不会知道在路途的具体地点。
